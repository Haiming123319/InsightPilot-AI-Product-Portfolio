# Bad Case 记录

## Case 1：字段识别错误

用户说“费用”时，系统可能在 `amount`、`budget_amount`、`approved_amount` 中选错字段。

改进方向：字段白名单、字段含义展示、低置信度时请求用户确认。

## Case 2：同比除零

上一年度费用为 0 时，直接计算同比会得到无穷大或误导性百分比。

改进方向：输出“新增费用”，禁止展示无意义百分比。

## Case 3：异常值影响结论

极端大额费用可能改变部门排名。

改进方向：展示含异常值和排除异常值两版结果，由用户确认口径。

## Case 4：图表与文字不一致

如果图表和结论不是同一份结构化结果生成，可能出现排名不一致。

改进方向：图表和结论共用 Python 计算结果，发布前做一致性校验。

## Case 5：不存在字段

模型可能引用数据中不存在的字段。

改进方向：执行前做字段合法性校验。

## 当前批量评测 Bad Case

### TC001：描述性统计未覆盖

问题：各部门总费用是多少。

当前表现：系统请求用户改为 2024/2025 部门费用同比分析，未直接执行普通分组求和。

改进方向：增加描述性统计意图、通用 groupby sum 工具和对应结果模板。

### TC005：异常分析未覆盖

问题：哪些记录属于异常大额费用。

当前表现：系统请求用户改为 2024/2025 部门费用同比分析，未返回异常明细。

改进方向：增加异常检测工具，支持 IQR/Z-score 阈值，输出异常记录、金额、部门、费用类型和判断依据。
