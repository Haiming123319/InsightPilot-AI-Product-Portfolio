# Bad Case 记录

更新时间：2026-07-13

## 当前状态

当前批量评测文件 `docs/evaluation_results.csv` 中 32 条用例均为 passed，`docs/bad_cases_log.csv` 当前为空。

本文件保留历史 Bad Case 和后续仍需观察的风险，用于作品集复盘。

## 历史 Bad Case 1：描述性统计未覆盖

问题：用户问“各部门总费用是多少”。

历史表现：系统只支持 2024/2025 部门同比分析，请求用户改成同比口径，无法直接回答普通分组求和问题。

修复动作：

- 增加 `department_summary` 意图类型
- 增加部门费用汇总工具
- 增加部门总额、占比和排名
- 增加部门费用柱状图
- 增加汇总结果校验

当前状态：已修复。TC001、TC007、TC008、TC009、TC023、TC031 均已通过。

## 历史 Bad Case 2：异常分析未覆盖

问题：用户问“哪些记录属于异常大额费用”。

历史表现：系统只做数据体检中的格式问题检测，没有面向用户问题返回异常金额明细。

修复动作：

- 增加 `anomaly_detection` 意图类型
- 增加 IQR 异常阈值计算
- 增加异常记录明细表
- 增加最大异常金额指标
- 增加异常值分布图
- 增加异常阈值一致性校验

当前状态：已修复。TC005、TC017、TC018、TC019、TC028、TC032 均已通过。

## 历史 Bad Case 3：不存在字段

问题：用户按客户、行业、预算、供应商、合同编号等不存在字段提问。

风险：如果系统强行执行，可能误用现有字段并生成错误分析。

修复动作：

- 增加字段白名单和缺失字段拦截
- 对不存在字段返回澄清问题
- 在评测集中加入 6 条错误字段用例

当前状态：已修复。TC006、TC020、TC021、TC022、TC029、TC030 均已通过。

## 仍需观察的风险

### 风险 1：真实 LLM 泛化

当前意图解析是规则版模拟，不代表真实 LLM 接入后的表现。

后续动作：接入真实 OpenAI API 后，比较纯 LLM、LLM + Python、LLM + Python + 校验三种方案。

### 风险 2：异常值影响同比结论

当前能识别异常大额费用，但还没有支持“排除异常值后重新分析”的多轮追问。

后续动作：增加追问状态和排除异常值后的重新计算。

### 风险 3：用户无法编辑计划

当前用户只能确认执行，不能删除、修改或新增计划步骤。

后续动作：增加计划步骤编辑能力，支持字段映射和分析口径调整。

### 风险 4：标准答案还偏结构化

当前标准答案主要记录标准字段、操作和任务类型，还没有为每条用例记录完整数值答案。

后续动作：补充标准数值、标准排名、容差范围和不可接受表达。
