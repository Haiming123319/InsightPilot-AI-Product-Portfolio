# InsightPilot AI 产品经理作品集成稿

## 1. 项目封面

InsightPilot：可验证、可解释的 AI 数据分析助手。

关键词：AI Copilot、数据分析、Python 工具调用、结果校验、可追溯结论。

## 2. 一句话产品定义

面向不熟悉 Python、SQL 和复杂 BI 工具的业务人员，通过自然语言完成 Excel 费用数据分析，并提供公式、代码、图表和校验结果溯源。

## 3. 用户画像

核心用户是需要定期处理 Excel 费用报表的业务分析人员。他们熟悉业务含义，但不一定掌握复杂公式、Python 或 SQL。

典型任务包括部门费用汇总、同比变化、费用类型下钻、异常大额费用识别和分析报告撰写。

## 4. 当前流程痛点

业务人员通常需要手动清洗金额和日期、检查重复值、写公式、做数据透视表、画图并撰写结论。任何一步出错都会影响结论可信度。直接使用通用大模型又容易出现数字编造、字段误用和不可复核的问题。

## 5. 产品目标与非目标

目标是让用户获得可执行、可验证、可修改的数据分析结果。第一版聚焦单文件费用分析，不做多文件关联、权限系统、多 Agent、RAG 或企业部署。

## 6. 核心场景

用户上传费用明细表后提问：分析 2025 年各部门费用变化，找出同比增长最快的部门，并判断主要增长来自哪些费用类型。

系统先生成分析计划，经用户确认后调用 Python 清洗数据、计算同比、下钻费用类型、生成图表和结论，并展示校验结果。

## 7. MVP 范围

已支持 CSV/XLSX 上传、数据体检、自然语言提问、计划确认、Python 执行、图表、结论、公式和代码溯源、Markdown 报告导出、批量评测和 Bad Case 日志。

## 8. 功能架构

```text
Streamlit UI
  ↓
文件解析与数据画像
  ↓
规则版意图解析
  ↓
分析计划生成
  ↓
Python 工具层
  ├─ 部门费用汇总
  ├─ 部门同比分析
  └─ 异常大额费用识别
  ↓
结果校验层
  ↓
图表、结论、证据和报告导出
```

## 9. 用户流程

上传数据 → 数据预览 → 字段体检 → 输入问题 → 查看结构化意图 → 确认分析计划 → 执行分析 → 查看图表和指标 → 查看自动校验 → 导出 Markdown 报告。

## 10. 页面设计

当前 Demo 使用 6 个核心页面区域：数据预览、字段体检、问题汇总、分析任务、结果与溯源、批量评测。每个页面都围绕“让用户知道 AI 做了什么”设计。

## 11. 职责划分

| 任务 | LLM/规则 | Python | 规则校验 | 用户 |
|---|---:|---:|---:|---:|
| 理解问题 | 是 | 否 | 是 | 可修正 |
| 生成计划 | 是 | 否 | 是 | 确认 |
| 清洗数据 | 否 | 是 | 是 | 可复核 |
| 数值计算 | 否 | 是 | 是 | 否 |
| 图表生成 | 否 | 是 | 是 | 否 |
| 结论解释 | 是 | 否 | 是 | 复核 |

## 12. Agent 架构

MVP 使用单 Agent + 多工具 + 状态机。Phase 0 保留规则服务作为稳定基线，Phase 1 已将真实 OpenAI 接入意图理解入口；模型不计算数字、不生成 Python、不直接执行工具。

## 13. Prompt 与结构化输出

项目保留了三类 Prompt 草稿：意图解析、计划生成、结果解释。Phase 1 的真实模型使用 Pydantic 结构化输出，并在本地校验字段、操作和时间范围；不存在字段会进入澄清而不是执行。

## 14. 结果验证机制

项目实现字段合法性、数值一致性、排名、文字数字一致性、图表数据一致性、异常阈值一致性和因果表达限制校验。结论必须绑定 evidence_id。

## 15. 测试集与指标

当前测试集包含 32 条问题，覆盖描述性统计、对比、趋势、下钻、异常和错误字段。指标包括任务完成率、任务理解准确率、校验通过率、澄清率和平均耗时。

## 16. 当前评测结果

在样例费用数据上，规则基线的 32 条用例全部通过。Phase 1 新增 13 条解析路由、失败、脱敏和结构化输出测试，自动化测试总数为 29 条。以上结果证明规则流程和接入边界可控，但还不能代表真实 LLM 泛化能力、成本或线上延迟。

## 17. 复盘与迭代

本项目的核心取舍是先用规则版意图解析跑通“AI 产品流程 + Python 工具 + 结果校验”，再接入真实模型。这样可以把不确定性隔离在意图解析层，保证数据计算和结论验证可控。

下一步进入 Phase 2，让用户确认后的分析计划真实驱动白名单工具；随后再比较纯 LLM、LLM + Python、LLM + Python + 校验三种方案，并录制 2-4 分钟 Demo。
