# 评测计划

## 评测对象

后续阶段比较三种方案：

| 方案 | 说明 |
|---|---|
| 纯 LLM | 直接让模型根据数据摘要回答 |
| LLM + Python | 模型理解和规划，Python 计算 |
| LLM + Python + 校验 | 在 Python 计算后加入规则校验 |

## 指标

- 任务理解准确率
- 数值准确率
- 任务完成率
- 图表正确率
- 结论一致率
- 幻觉率
- 人工修改率
- 响应时间
- 单次成本

## 当前真实评测结果

测试数据：`data/sample_expenses_dirty.csv`

测试集：`data/test_cases.csv`

导出文件：

- `docs/evaluation_results.csv`
- `docs/bad_cases_log.csv`

| 指标 | 当前结果 |
|---|---:|
| 用例数 | 6 |
| 任务完成率 | 66.7% |
| 校验通过率 | 66.7% |
| 澄清率 | 50.0% |
| 平均耗时 | 约 4.3 ms |

当前通过的用例：

- TC002：哪个部门 2025 年同比增长最快
- TC003：市场部费用按月如何变化
- TC004：市场部增长主要来自哪些费用类型
- TC006：按客户行业分析费用变化，正确请求澄清

当前失败的用例：

- TC001：各部门总费用是多少。当前系统只支持 2024/2025 同比分析，未实现普通描述性统计工具。
- TC005：哪些记录属于异常大额费用。当前已能检测数据质量问题，但还未实现面向用户问题的异常明细分析工具。
