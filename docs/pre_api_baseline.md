## Phase 0 风险清单

| 风险 | 事实证据 | 产品影响 |
|---|---|---|
| 真实模型未进入主流程 | 页面原先直接调用 `parse_user_intent` | 不能证明模型能理解开放式业务问题 |
| 规则解析与模型解析没有统一元数据 | 原先只返回 `AnalysisIntent` | 无法比较 Provider、延迟、Token 和回退体验 |
| 数据画像不包含完整模型输入约束 | 原先只传字段名 | 模型无法基于字段类型、时间范围做可靠判断 |
| 分析计划尚未驱动执行 | `task_service.py` 按 `task_type` 分支 | 计划编辑的产品价值尚未被证明，留到 Phase 2 |
| 结果解释仍是模板 | `task_service.py` 直接构造 `summary` 和 `claims` | 当前不能宣称已实现真实模型解释，留到 Phase 3 |

## 基线保留策略

- 规则解析继续作为默认 Provider。
- 不删除固定 Python 分析函数、模板解释和现有评测文件。
- 所有外部 API 测试使用 Mock，不在 `pytest` 中消耗真实 API。
- Phase 1 只改变意图理解入口，不改变数值计算、图表和结果校验逻辑。
